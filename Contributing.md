# Contributing to Sarkari Saathi

We enforce strict engineering standards. 

## Branching Strategy
* `main`: Production-ready code. Do not push directly to main.
* `feat/feature-name`: For new features (e.g., `feat/vector-search`).
* `fix/bug-name`: For bug fixes.

## Pull Request Process
1. Fork the repository and create your branch from `main`.
2. Ensure your code passes all linting (`npm run lint`).
3. Update documentation for any architecture changes.
4. Submit a PR with a detailed description of the payload and logic changes.
